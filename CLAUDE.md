# CLAUDE.md

## 카파시 4원칙 (작업 시 반드시 준수)

Andrej Karpathy가 정리한 LLM 코딩의 실패 모드에서 나온 네 가지 원칙이다. 이 저장소에서
코드를 만지는 에이전트는 아래를 기본 규칙으로 삼는다.

1. **Think Before Coding — 짜기 전에 생각한다.**
   모호하면 멋대로 하나를 골라 달리지 말고, 가정을 먼저 말하고 필요하면 물어본다. 해석이
   갈리는 지점은 드러내 놓고 시작한다.

2. **Simplicity First — 단순함이 먼저다.**
   지금 요청된 것만 만든다. 미래의 가상 요구를 위한 추상화, 안 물어본 예외 처리, 교과서식
   디자인 패턴을 미리 끌어오지 않는다.

3. **Surgical Changes — 필요한 곳만 고친다.**
   건드릴 곳만 정확히 건드리고 주변 스타일을 따른다. 요청하지 않은 리팩터링으로 무관한
   코드를 흔들지 않는다.

4. **Goal-Driven Execution — 목표를 세우고 확인될 때까지 돈다.**
   "대충 됐다"에서 멈추지 않는다. 검증 가능한 성공 기준을 정하고, 그 기준을 만족할 때까지
   실행·확인을 반복한다.

---

## 이 저장소가 뭔지

MCP 툴 라우팅의 recall 절벽을 측정하는 오프라인 벤치마크다. 툴 카탈로그가 수백 개로 커질 때
semantic top-k 라우팅이 정답 툴을 놓치는 현상을, 합성 카탈로그와 mock 임베딩으로 재현하고
실제 임베딩(bge-small)으로 교차검증한다. 서빙 게이트웨이가 아니라 평가 하네스다.

## 실행

```bash
make bench       # 오프라인·순수 stdlib·결정적. artifacts/ 생성
make test        # unittest 21케이스
make sweep       # core_share 민감도 스윕
make bench-real  # bge-small 실제 임베딩 (pip install .[local] 필요)
```

## 손대기 전에 알아야 할 것

- **기본 경로는 순수 stdlib다.** 서드파티 의존성을 코어에 새로 넣지 않는다. 무거운 것은
  전부 `providers/`·`.[extra]` 뒤의 opt-in으로 둔다.
- **결정성은 계약이다.** 같은 seed면 mock 경로의 모든 수치가 바이트 단위로 같아야 한다.
  `Math.random`/`time` 기반 값을 로직에 넣지 않는다(측정 latency 같은 비결정 값은 리포트에서 뺐다).
- **mock과 real을 섞지 않는다.** 헤드라인 수치가 mock인지 실제 모델인지 항상 명시한다.
  없는 실행 증거를 있는 것처럼 쓰지 않는다.
- **과설계를 되돌리지 않는다.** pgvector·별도 LangGraph 에이전트·OTel·latency 파이프라인은
  의도적으로 뺐다. 다시 넣지 않는다(원칙 2).
